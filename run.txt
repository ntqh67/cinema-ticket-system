QUY TRÌNH CHẠY VÀ BÀN GIAO DATABASE
====================================

Quy ước:
- Nhánh daubuoi chỉ dùng để nhận và bàn giao file dump PostgreSQL.
- Nhánh main dùng để chạy và phát triển code.
- Chỉ một người được cập nhật dữ liệu database tại một thời điểm.
- Tất cả lệnh dưới đây chạy bằng PowerShell tại thư mục gốc dự án.


I. NHẬN DATABASE MỚI NHẤT
=========================

CẢNH BÁO:
Lệnh import sẽ thay dữ liệu PostgreSQL hiện tại bằng dữ liệu trong dump.
Nếu máy đang có dữ liệu chưa bàn giao thì phải export và push trước.

git switch daubuoi
git pull --ff-only origin daubuoi

docker compose up -d postgres redis

Get-Content -Raw .\database\cinema_ticket_system_dump.sql |
  docker compose exec -T postgres psql -v ON_ERROR_STOP=1 -U postgres -d cinema_ticket_system

npx prisma generate

Sau khi import:
- KHÔNG chạy: npx prisma db seed
- KHÔNG chạy: npx prisma migrate reset
- KHÔNG chạy lại import nếu chỉ muốn bật dự án vào lần sau.


II. CHUYỂN VỀ NHÁNH CODE
========================

git switch main
git pull --ff-only origin main

Nếu package.json vừa thay đổi:

npm install
npm --prefix backend install
npx prisma generate


III. CHẠY BACKEND - TERMINAL 1
==============================

cd backend
npm run start:dev

API:     http://localhost:3000/api
Swagger: http://localhost:3000/api/docs


IV. CHẠY FRONTEND - TERMINAL 2
==============================

cd frontend
npx vite --host 0.0.0.0 --port 5173

Frontend: http://localhost:5173


V. NHỮNG LẦN CHẠY SAU
======================

Không cần import lại dump. Chỉ chạy:

docker compose up -d

Sau đó chạy backend và frontend ở hai terminal như mục III và IV.

Không chạy docker compose down -v vì lệnh này xóa volume database.


VI. BÀN GIAO DỮ LIỆU
====================

Trước khi bàn giao:
- Dừng nhập dữ liệu và bảo đảm không còn request đang ghi database.
- Code đang sửa trên main phải được commit hoặc xử lý riêng trước khi đổi nhánh.
- Không có người thứ hai cùng cập nhật database.

Từ thư mục gốc dự án:

git switch daubuoi
git pull --ff-only origin daubuoi

powershell -ExecutionPolicy Bypass -File .\scripts\export-database-dump.ps1

git add database\cinema_ticket_system_dump.sql
git diff --cached --check
git commit -m "chore: update database dump"
git push origin daubuoi

Sau khi push thành công, báo cho người tiếp theo nhận lượt database.

Nếu cần tiếp tục chạy code sau khi bàn giao:

git switch main
git pull --ff-only origin main


VII. TÓM TẮT LUỒNG LÀM VIỆC
===========================

Người A:
nhận dump -> chuyển main -> chạy code -> nhập dữ liệu -> chuyển daubuoi
-> export dump -> commit -> push -> bàn giao

Người B:
chờ A bàn giao -> pull daubuoi -> import dump -> chuyển main -> tiếp tục làm

Dữ liệu doanh thu, booking, thanh toán và vé chỉ được bàn giao nếu chúng đã
nằm trong PostgreSQL trước lúc chạy export dump.
