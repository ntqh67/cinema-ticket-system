HUONG DAN CHAY SEPAY GIONG MAY QUANG HUY
========================================

Muc dich
--------

chay luong thanh toan SePay trong moi truong local bang ngrok.

Luu y bao mat
-------------
Khong commit file .env that len GitHub.
Moi nguoi tu copy .env.example thanh .env va dien thong tin rieng.
SEPAY_API_KEY va ngrok authtoken la secret, khong dua vao tai lieu public.

7. Kiem tra ngrok noi duoc backend
----------------------------------
Mo trinh duyet:

https://xxxxx.ngrok-free.dev/api/bookings/payment-methods

Ket qua dung la JSON co:

"id": "sepay", "enabled": true, "mode": "live"

Neu vao / bi "Cannot GET /" thi khong sao, vi backend khong co route goc.
Can test dung route /api/bookings/payment-methods.

8. Cau hinh webhook tren SePay
-----------------------------
Trong SePay Dashboard, them webhook:

Ten webhook:
CRTicket Booking Payment

URL nhan webhook:
https://xxxxx.ngrok-free.dev/api/bookings/sepay-webhook

Loai giao dich:
Tien vao

Dinh dang du lieu:
JSON

Bao mat:
API Key

API key:
Dung gia tri SEPAY_API_KEY trong .env.

Neu SePay cho nhap header day du, dung:

Authorization: Apikey <SEPAY_API_KEY>

9. Theo doi webhook
-------------------
Mo:

http://127.0.0.1:4040

Khi SePay goi dung, se thay request:

POST /api/bookings/sepay-webhook

Y nghia status:
- 200: backend da nhan webhook.
- 401: sai API key hoac SePay gui header khac voi backend mong doi.
- Khong co request: SePay chua goi webhook hoac URL webhook dang sai.
- 200 nhung payment khong thanh cong: kiem tra payment_webhook_events,
  thuong la sai so tien, sai noi dung chuyen khoan hoac booking da het han.

10. Luong test thanh cong
-------------------------
1. Dang nhap tren web.
2. Chon phim.
3. Chon suat chieu.
4. Chon ghe.
5. Chon hoac bo qua combo bap nuoc.
6. Vao thanh toan.
7. Chon SePay QR.
8. Bam xac nhan thanh toan.
9. Quet QR va chuyen khoan dung:
   - dung so tien;
   - dung noi dung chuyen khoan dang hien, vi du CRT...
10. SePay gui webhook ve backend qua ngrok.
11. Frontend tu polling /api/bookings/:bookingId/payment-status.
12. Khi payment SUCCESS, web tu chuyen sang trang ve dien tu.

Ket qua dung trong database:
- payments.status = SUCCESS
- bookings.status = PAID
- showtime_seats.status = BOOKED
- tickets co ve moi
- payment_webhook_events co log webhook PROCESSED

11. Test nhanh bang webhook gia
-------------------------------
Dung khi khong muon chuyen khoan that hoac can test logic backend.
Copy so tien va noi dung CRT dang hien tren QR roi chay:

$headers = @{ Authorization = "Apikey <SEPAY_API_KEY>" }
$body = @{
  id = "TXN-TEST-001"
  transferType = "in"
  transferAmount = 180000
  content = "Thanh toan CRT_MA_DANG_HIEN_TREN_QR"
} | ConvertTo-Json

Invoke-RestMethod `
  -Uri "http://localhost:3000/api/bookings/sepay-webhook" `
  -Method Post `
  -Headers $headers `
  -ContentType "application/json" `
  -Body $body

Phai thay:
- transferAmount bang dung so tien tren QR.
- content co dung ma CRT dang hien tren QR.

12. Loi thuong gap
------------------
Web dung o "Dang cho SePay xac nhan":
- Mo http://127.0.0.1:4040.
- Neu khong co POST /api/bookings/sepay-webhook: SePay chua goi webhook.
- Neu co 401: sai API key/header.
- Neu co 200 nhung khong ra ve: xem payment_webhook_events va status payment.

Ngrok URL doi:
- Moi lan ngrok doi URL, phai cap nhat lai webhook URL tren SePay.

Booking het han:
- Mac dinh giu booking 10 phut.
- Het han thi booking/payment thanh EXPIRED, can tao booking moi.
